# 阶段二前置：提升旋转识别置信度规划

## 背景

阶段一 OpenCV 原型已经能在前 5 张样例上得到正确角度：

- 001：270 度
- 002：270 度
- 003：180 度
- 004：180 度
- 005：180 度

但第 2 张置信度只有 0.1217，被标记为需要人工复核。当前置信度低的主要原因是算法按整条页面边带计算表格线密度。标题栏实际只占边带局部区域，整边评分会被以下因素影响：

- 标题栏之外的大面积空白稀释密度。
- 零件轮廓线、尺寸线进入边带。
- 明细表或技术要求表可能与标题栏竞争。
- 当前置信度只看第一名与第二名分差，证据维度单一。

## 取经结论

相关资料调研见：

- `references/drawing-rotation-research/README.md`
- `docs/research/2026-06-23-engineering-drawing-orientation-research.md`

调研后的关键修正：

- OpenCV 形态学线条提取适合作为底层特征，但评分对象应是局部候选框，而不是整条边。
- 工程图纸 OCR 容易受线条、符号和布局影响，适合作为第二层确认，不适合作为第一层方向判断。
- 近期工程图纸解析方案倾向先检测标题栏/关键区域，再抽取信息；当前项目应先实现轻量级局部候选框检测。
- 置信度应解释候选框自身质量和竞争候选歧义，不应只是把数值调高。

## 目标

在不引入 OCR 和视觉 MCP 的前提下，优先提高 OpenCV 层的置信度质量。

目标不是盲目把数值调高，而是让置信度更接近“判断是否可靠”的真实含义。

## 总体策略

将阶段一的“整边评分”升级为“局部标题栏候选评分”：

1. 在四个边缘区域内寻找局部表格候选框。
2. 对候选框而不是整条边带打分。
3. 加入符合标题栏形态的几何特征。
4. 用多证据融合计算置信度。
5. 保留低置信度人工复核机制。

## 具体规划

### 1. 表格候选框提取

基于水平线、垂直线和交点图，寻找连通区域或轮廓候选。

候选框过滤条件：

- 面积不能过小。
- 宽高比符合标题栏在当前旋转状态下的形态。
- 水平线和垂直线都要存在。
- 交点数量达到最低阈值。
- 候选框需靠近页面边缘。

### 2. 标题栏形态评分

每个候选框计算以下特征：

- `line_density`：线条密度。
- `intersection_density`：横竖线交点密度。
- `grid_balance`：水平线和垂直线数量是否均衡。
- `edge_proximity`：距离最近页面边缘的接近程度。
- `corner_preference`：是否靠近标题栏常见角落。
- `size_reasonableness`：尺寸是否接近标题栏常见比例。
- `local_contrast`：候选框内部线条密度是否明显高于周围区域。

### 3. 边侧聚合

不再直接对整条边打分，而是：

- 每个边侧选取得分最高的 1 到 3 个候选框。
- 用最高候选分作为该边主分。
- 用候选数量和候选集中程度作为辅助分。

### 4. 置信度计算

置信度由多项证据组成：

- 第一名与第二名候选分差。
- 第一名候选框自身质量。
- 是否符合标题栏几何形态。
- 是否存在强竞争候选。
- 是否触发经验规则覆盖。

建议输出：

- `confidence`：0 到 1。
- `evidence_score`：候选框自身证据强度。
- `margin_score`：与第二名差距。
- `ambiguity_score`：竞争候选带来的歧义。
- `needs_review`：是否需要人工复核。

### 5. 调试输出增强

调试图应从“画四条边带”升级为：

- 画出所有候选标题栏框。
- 高亮最终候选框。
- 标注候选分数和所属边。
- 可选输出中间图：
  - 二值图
  - 水平线图
  - 垂直线图
  - 交点图

## 验收标准

先以当前前 5 张为基准样例，再扩展到前 20 张实验样本：

- 识别角度保持正确：
  - 001：270 度
  - 002：270 度
  - 003：180 度
  - 004：180 度
  - 005：180 度
- 第 2 张置信度应明显提升，目标不低于 0.25。
- 不应通过硬编码文件名或页码提升置信度。
- 若存在强竞争候选，仍允许标记 `needs_review=true`，但需要给出更清晰的证据字段。
- 调试图能直观看到最终候选标题栏框，而不只是整条边带。
- 全量 63 张实验样本应先完成抽取和渲染，用于检验算法是否泛化并支持阈值校准。

## 暂不做

本次规划暂不引入：

- OCR 字段识别。
- 视觉 MCP 兜底。
- PDF 自动校正。
- 全 63 页批量处理。

这些应在 OpenCV 层局部候选评分稳定后再进入下一阶段。

# OpenCV 阶段二高置信误判修复计划

## 背景

全量 63 张图纸三方比对后发现，当前 OpenCV 阶段一不是单纯低置信问题，而是存在高置信误判：

| 样本 | OpenCV | MCP | 人工复核 | 问题 |
| --- | --- | --- | --- | --- |
| `sample_009` | 0 度 | 270 度 | 270 度 | 底部整边带表格干扰压过右侧标题栏 |
| `sample_010` | 270 度 | 0 度 | 0 度 | 右侧整边带表格干扰压过底部标题栏 |
| `sample_042` | 270 度 | 270 度 | 270 度 | 方向正确但置信度低，应继续复核 |

## 目标

阶段二目标是降低高置信误判，而不是简单提高或降低置信度：

- `sample_009` 应从 0 度修正为 270 度。
- `sample_010` 应从 270 度修正为 0 度。
- `sample_042` 可保持 270 度并继续标记复核。
- 全量 63 张结果与当前三方候选真值不应新增分歧。

## 技术路线

1. 保留现有线条提取、候选框提取和调试图输出。
2. 在候选标题栏评分中加入“标题栏形态”证据：
   - 右侧标题栏通常是靠右的竖向窄长表格。
   - 底部标题栏通常是靠下的横向宽表格。
   - 横竖比例与对应边不匹配时降低候选可信度。
3. 将最终方向选择从“整边带分数优先”改为“候选框优先，整边带兜底”：
   - 若局部候选框质量足够，优先采用局部候选框所在边。
   - 若局部候选框缺失或质量过低，再退回整边带评分。
4. 对 009/010/042 做回归检查，再跑全量 63 张。

## 验收标准

- `sample_009`、`sample_010` 与人工复核一致。
- 三方比对分歧中不再出现 OpenCV 对 009/010 的错误。
- 全量输出仍包含 `confidence`、`needs_review`、`best_candidate`、`side_scores`、`candidates`。
- 图纸、PNG、PDF 和输出目录继续保持在 `.gitignore` 覆盖范围内。

## 回滚

实现前提交本计划、RPD 和 TODO。若阶段二算法效果变差，可回退到该提交，保留已确认的实验结论。

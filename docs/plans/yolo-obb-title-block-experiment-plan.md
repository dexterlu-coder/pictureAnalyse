# YOLO/OBB 标题栏检测小实验计划

## 背景

当前 OpenCV 主流程在原始人工确认集和 90 度增强集上的联合评估为 83/83。它适合作为基线继续保留，但不应在没有新失败样本的情况下继续硬调。

下一步主线调整为：

```text
OpenCV 基线
  -> 本地 YOLO/OBB 标题栏检测
  -> OCR 文字证据
  -> 本地/云端 VLM 兜底解释疑难样本
  -> 证据融合，不做简单同权投票
```

该路线的核心判断是：标题栏检测比 OCR 和 VLM 更接近当前任务本身。我们要识别的是标题栏位置，而不是理解整张图纸。

## 目标

验证一个本地 `title_block` 检测器是否能稳定找到机械图纸标题栏，并为旋转角度判断提供 OpenCV 之外的强证据。

本阶段回答三个问题：

1. YOLO/OBB 是否能在本地样本上稳定检测标题栏框。
2. 标题栏框位置是否能可靠映射为 `bottom/left/top/right`。
3. YOLO/OBB 与 OpenCV 冲突时，是否能提供有价值的复核证据。

## 非目标

本阶段不做：

- 不上传图纸到外部服务。
- 不接入云端 VLM。
- 不训练大模型。
- 不将 OCR 作为主方向识别器。
- 不替换 OpenCV 主流程。
- 不在标注规范确认前开始训练。

## 输入样本

本地忽略目录中的 83 张实验图片：

- 原始人工确认集：`local_data/experiment_samples/all/png/`，共 63 张。
- 顺时针 90 度增强集：`local_data/experiment_samples/augmented_90/png/`，共 20 张。

ground truth：

- `local_data/ground_truth/rotation_ground_truth.json`
- `local_data/ground_truth/rotation_ground_truth_augmented_90.json`

## 标注规范草案

类别只设一个：

```text
0 title_block
```

标注对象：

- 只标注完整标题栏外框。
- 若标题栏与明细表相连，只圈标题栏主体，不把整张明细表一起框入。
- 若标题栏在右下方，仍按实际标题栏外边界标注，不人为扩成整条右边或下边。
- 若标题栏因扫描裁切缺边，按可见边界估计最小外接旋转框，并在备注中记录 `cropped`。
- 若图纸有多个类似表格，只标注符合标题栏字段结构的区域。

OBB 标签格式采用 Ultralytics OBB 数据集格式：

```text
class_index x1 y1 x2 y2 x3 y3 x4 y4
```

其中 4 个点为归一化图像坐标。正式训练前需要用 2 到 3 张样例标注做加载检查，确认点序和可视化结果正确。

## 数据拆分策略

样本数量较少，因此分两阶段：

### A. 管线冒烟实验

- 先标注 12 到 20 张。
- 覆盖 `right/top/bottom/left` 四类标题栏位置。
- 目的不是追求泛化，而是验证标注格式、训练命令、预测输出和可视化链路。

### B. 83 张小样本实验

- 标注全部 83 张。
- 使用按来源分组的拆分，避免同一源图的增强版和原图同时泄漏到训练与验证中。
- 优先将 `sample_009`、`sample_010`、`sample_042` 和 `aug90_016_from_sample_042` 放入验证或测试关注集。

建议拆分：

- `train`：约 70%。
- `val`：约 15%。
- `test_focus`：约 15%，保留疑难和代表性样本。

## 实验步骤

1. 生成本地标注准备包：
   - 样本清单。
   - 图片路径。
   - 当前人工确认的粗位置和精确位置。
   - 建议拆分。
   - 待标注状态。
2. 人工标注标题栏 OBB 框。
3. 将标注转换为 Ultralytics OBB 数据集结构。
4. 使用 YOLO OBB nano/small 模型做训练冒烟实验。
5. 对验证集输出：
   - 检测框。
   - 置信度。
   - 框中心所在边。
   - 映射旋转角度。
6. 与 OpenCV 和人工 ground truth 比对。
7. 记录是否适合进入证据融合流程。

## 调试协议

根据 YOLO/OBB 调试方案他山调研，正式训练前必须增加以下质量门：

### 标注前

- 冒烟集必须覆盖 `left/right/top/bottom`。
- 同源原图和增强图不得同时泄漏到 train 与 val/test_focus。
- 标注工具必须能导出或转换为 Ultralytics OBB 格式。

### 标注后

- 必须生成 OBB label overlay 调试图。
- 必须检查 4 个归一化点是否都在 `[0, 1]` 范围内。
- 必须检查是否误框明细表、技术要求表或非标题栏表格。
- 如发现规则歧义，先修改标注规范，再继续标注。

### 训练冒烟

- 第一轮训练只验证链路，不追求泛化。
- 若训练集也学不会，优先查标签格式、点序、类别 YAML 和标注一致性。
- 若训练集能学会但验证集差，再考虑补样本、改拆分或调增强。

### 错误分层

每个失败样本必须归因到：

- `label_error`
- `format_error`
- `false_negative`
- `false_positive`
- `localization_error`
- `postprocess_error`
- `data_leakage`

详细调研见 `docs/research/2026-06-25-yolo-obb-debugging-research.md`。

## 验收标准

冒烟实验通过标准：

- 数据集能被 YOLO/OBB 正常加载。
- 预测结果能在调试图中画出旋转框。
- 至少能在小样本上稳定检测到标题栏。

83 张实验通过标准：

- 标题栏检测召回率达到可观察的高水平，优先目标为 95% 以上。
- 检测框映射的标题栏粗位置与人工 ground truth 基本一致。
- 对 OpenCV 已知关注样本能输出可解释证据。
- 未检测或低置信样本必须进入人工复核，不得静默接受。

## OCR 的位置

OCR 不提前于 YOLO/OBB 做主线实验。它的角色是标题栏真实性确认：

- 是否识别到图号、名称、材料、比例、日期、审核、单位等字段。
- 文字框是否集中在 YOLO/OBB 检测到的标题栏区域。
- 文字方向是否支持当前旋转判断。

OCR 的结果只作为辅助证据，不作为主方向判断器。

## VLM 的位置

VLM 暂时不抢主线。它进入流程的条件是：

- OpenCV 与 YOLO/OBB 冲突。
- YOLO/OBB 无检测或低置信。
- OCR 证据不足。
- 用户需要解释非典型版式。

第一轮 VLM 价值验证可用云端模型降低复杂性；本地 VLM 后续再按配置压力逐级测试。

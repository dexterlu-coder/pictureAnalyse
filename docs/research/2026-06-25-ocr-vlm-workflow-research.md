# OCR + VLM 工作流取经笔记

日期：2026-06-25

## 一、学习目标

- 我们真正要解决的问题：机械图纸旋转识别已经在当前样本上稳定，需要规划 OCR 和图像识别大模型如何加入工作流。
- 本轮不做的事情：不接入智谱、阿里或其他外部 API；不上传图纸；不改现有 OpenCV 主流程。
- 最终沉淀物：样本索引、横向取经笔记、OCR/VLM 兜底流程规划。

## 三、对象模型

这个领域的核心对象不是“调用几个模型”，而是：

- `ProviderResult`：每个识别器输出统一结构，包括位置、角度、置信度和证据。
- `Evidence`：几何证据、OCR 关键词证据、VLM 视觉判断证据。
- `Gate`：决定是否接受 OpenCV、是否触发 OCR/VLM、是否进入人工复核。
- `FusionDecision`：融合后的最终判断，不是简单多数票。
- `AuditRecord`：保存每个 provider 的原始输出、融合理由和人工修正。

## 四、工作协议

1. 前置判断：先跑 OpenCV，因为当前联合评估已达到 83/83，成本最低且可解释。
2. 中间产物：OpenCV 输出候选、置信度、竞争候选和 debug 图。
3. 人机分工：高置信自动通过；冲突或低置信进入 OCR/VLM 兜底；仍冲突才给人。
4. 执行方式：OCR 与 VLM 只在触发条件满足时并行执行。
5. 审核与打回：融合结果必须记录证据；不能只输出最终角度。
6. 进化沉淀：人工纠正要写回 ground truth 和规则文件。

## 五、横向对照

| 模式 | 强项 | 弱项 | 可迁移原则 | 不迁移 |
| --- | --- | --- | --- | --- |
| 全量串行：OpenCV -> OCR -> VLM | 简单、容易追踪 | 前一步错会带偏后一步 | 中间产物可追踪 | 不把后续步骤限制在错误候选框内 |
| 全量并行投票 | 覆盖面大 | 成本高、慢、模型间置信度不可比 | 多源证据有价值 | 不做一人一票 |
| 多阶段门控 | 成本低、可解释、适合生产 | 阈值和触发规则要维护 | 主流程 + 条件兜底 | 不把 VLM 当默认入口 |
| 混合置信度层 | 可校准自动化边界 | 实现复杂 | 置信度要独立评估 | 当前不引入重型统计模型 |

## 六、不可变原则

1. OpenCV 继续做主识别器，除非联合评估证明它失效。
2. OCR 和 VLM 是证据补强，不是默认替代，且二者不等价。
3. 触发兜底的依据必须是可审计的：低置信、候选竞争、规则冲突、抽检策略。
4. 融合不是简单投票，而是按证据类型加权。
5. 大模型输出必须结构化，并保留原文证据。
6. 涉及图纸数据时，默认最小化外发，必要时才调用外部 API。
7. 若需要第三类识别器，优先考虑专用标题栏检测模型或专用方向分类器，而不是再增加一个通用 VLM。

## 七、我们的工作流骨架

```text
PDF/PNG
  -> OpenCV 主识别
  -> 质量门
      -> 高可信：直接接受
      -> 低可信/冲突：
            -> OCR 分支
            -> 专用标题栏/方向模型分支
            -> VLM 分支（智谱/阿里等）
            -> 证据融合
                -> 一致或强证据：输出
                -> 仍冲突：人工复核
```

统一 provider 输出：

```json
{
  "provider": "opencv",
  "title_block_position": "right_top",
  "rotation_degrees": 270,
  "confidence": 0.82,
  "evidence": [
    "right-side local candidate score 47.7",
    "edge proximity 1.0"
  ],
  "needs_review": false
}
```

## 八、下一轮验证

- 用当前 83 张联合评估集做 OCR 小实验。
- 优先选样本：`sample_042`、`sample_009`、`sample_010`、`aug90_016_from_sample_042`。
- 成功标准：OCR 能在标题栏附近识别出图号、日期、公司名、材料、审核等关键词，并能辅助判断文字方向或标题栏位置。
- 失败点：扫描太淡、旋转后 OCR 文字方向识别不稳定、OCR 结果与 OpenCV 冲突但无法解释。
- 另一路验证：评估专用标题栏检测/方向分类器是否比 OCR 更适合作为第三类方向证据。

# 机械图纸旋转方向识别规划

## 背景

当前 PDF 中包含多张机械图纸扫描件。图纸符合机械制图规范，标题栏在正确方向下应位于页面下方或右下方。扫描件可能被随机顺时针旋转 0、90、180 或 270 度。

已有规则文件：

- `rules/mechanical-drawing-rotation.md`

## 目标

构建一个可逐步验证的自动识别流程，用于判断每张图纸当前相对于正确方向被顺时针旋转了多少度，并给出校正建议。

输出字段建议包括：

- 文件名或页码
- 标题栏候选位置
- 已顺时针旋转角度
- 建议校正角度
- 置信度
- 是否需要人工复核
- 调试图路径

## 推荐技术路线

采用混合方案：

1. OpenCV 图像处理定位标题栏候选区域。
2. OCR 辅助确认候选区域是否包含标题栏字段。
3. 视觉 MCP 只作为低置信度兜底复核。

## 阶段规划

### 阶段零：公开仓库整理

在推送到 GitHub public 仓库前，先整理项目目录并排除敏感图纸资料。

主要任务：

- 新增 `.gitignore`，忽略图纸原件、拆分页、渲染图、临时目录和本地草稿。
- 将当前图纸原件、拆分 PDF、渲染 PNG 移入 `local_data/` 下的本地私有目录。
- 保留公开仓库所需的规则、规划、RPD、TODO 和后续源码。
- 推送前使用 `git status --short` 确认没有敏感 PDF/PNG 被跟踪或待提交。

推荐本地私有目录：

- `local_data/source_pdfs/`
- `local_data/experiment_samples/all/`
- `local_data/experiment_samples/first20/`

### 阶段一：OpenCV 原型

只处理前 5 张已拆分 PDF 或对应 PNG，不修改 PDF 文件。

主要任务：

- 将输入 PDF 页面渲染为 PNG。
- 灰度化、二值化。
- 检测水平线和垂直线。
- 找表格密集区域。
- 判断表格密集区域位于页面下、左、上、右哪一侧。
- 按规则文件映射旋转角度。
- 输出 JSON/CSV 结果。
- 生成调试图，框选候选标题栏区域。

验收标准：

- 前 5 张图纸的判断结果应与人工判断一致。
- 结果中能看出候选标题栏位置和置信度。
- 调试图能辅助人工确认算法判断依据。

### 阶段二：OCR 辅助确认

对阶段一找到的候选区域进行 OCR，查找标题栏常见字段，如图号、名称、材料、比例、日期、单位名称和签字栏。

OCR 仅用于提高置信度，不作为唯一判断依据。

### 阶段三：视觉 MCP 兜底

当 OpenCV 和 OCR 结果低置信度时，调用视觉 MCP 判断标题栏位置。

触发条件示例：

- 多个边缘区域都有强表格候选。
- 标题栏被裁切。
- OCR 失败。
- 扫描噪声导致表格检测不稳定。

### 阶段四：批量处理与校正

在识别流程稳定后，处理完整 PDF 的全部页面，并可选择生成校正后的单页 PDF 或汇总 PDF。

## 第一阶段详细计划

第一阶段只做识别，不做 PDF 校正。

输入：

- `local_data/experiment_samples/all/pdf/*.pdf`
- 或已渲染的 `local_data/experiment_samples/all/png/*.png`

输出：

- `outputs/rotation-detection/stage1/results.json`
- `outputs/rotation-detection/stage1/results.csv`
- `outputs/rotation-detection/stage1/debug/*.png`

算法步骤：

1. 读取每张 PNG；若只有 PDF，则先用 Ghostscript 渲染为 PNG。
2. 转灰度图。
3. 自适应阈值或 Otsu 二值化。
4. 使用形态学操作提取水平线和垂直线。
5. 合并线段，形成表格候选区域。
6. 计算每个候选区域的特征：
   - 面积
   - 线条密度
   - 水平线数量
   - 垂直线数量
   - 与页面四边的距离
   - 宽高比
7. 将页面分为下、左、上、右四个边缘区域。
8. 对每个边缘区域计算标题栏置信度。
9. 选出置信度最高的一侧作为标题栏位置。
10. 按 `rules/mechanical-drawing-rotation.md` 映射旋转角度。
11. 输出结构化结果和调试图。

阶段一暂停点：

- 在实现阶段一代码前，需先由用户审核本计划。

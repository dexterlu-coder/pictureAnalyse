# YOLO/OBB 调试方案他山调研计划

## 背景

用户提醒：在开始 YOLO/OBB 标题栏标注和训练前，应先调研类似工程中的 YOLO 调试方案。

该提醒是必要的。YOLO/OBB 实验的风险不只在模型能不能训练，还在：

- 标注是否被模型正确理解。
- 数据拆分是否泄漏。
- 训练是否只是记住小样本。
- 错误样本是否能被分层分析。
- 推理输出是否能可靠映射为标题栏位置。

## 调研目标

形成一套适用于本项目的 YOLO/OBB 调试协议，覆盖：

1. 数据集和标注检查。
2. 小样本过拟合冒烟实验。
3. 训练过程观察指标。
4. 推理结果可视化和错误分类。
5. OBB 检测框到标题栏位置的后处理检查。
6. 何时回到标注修正，何时调整模型或参数。

## 调研范围

优先参考：

- Ultralytics 官方文档和官方仓库资料。
- OBB 数据集格式和训练/验证/预测流程。
- CVAT 等标注工具官方文档中与旋转框相关的能力。
- 工程图纸/技术图纸领域中使用 YOLO、OBB 或 layout detector 的论文。

本轮不做：

- 不安装新训练依赖。
- 不启动训练。
- 不绘制真实 OBB 标签。
- 不接入云端 VLM。

## 输出

- 新增调研笔记：`docs/2026-06-25-yolo-obb-debugging-research.md`。
- 更新 `docs/yolo-obb-title-block-experiment-plan.md`，加入调试协议。
- 更新 RPD 和 TODO。

## 通过标准

- 明确标注前必须做哪些检查。
- 明确训练前的冒烟流程。
- 明确训练后看哪些产物。
- 明确失败时如何判断是数据、标注、模型还是后处理问题。

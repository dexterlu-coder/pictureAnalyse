# 人工复核与标注界面规则

## 适用范围

本规则适用于项目中所有需要人工查看、填写、复核或标注的文件与界面，包括：

- 人工复核 CSV。
- 人工复核 HTML。
- 标注任务清单。
- 图像查看索引。
- 后续 OCR/VLM/YOLO 调试人工审核包。

## 核心原则

最终结果稳定可靠是最高优先级。

当稳定可靠与速度、自动化程度、少走步骤发生冲突时，必须优先选择稳定可靠。对于机械图纸方向识别、标题栏检测、OCR/VLM 兜底和后续 PDF 校正等流程，不能为了更快进入下一步而跳过人工复核、可视化检查、数据校验或错误分层。

人工界面只呈现完成当前任务所必需的信息。

不应把模型内部字段、调试分数、冗余路径、长 JSON、候选列表、算法细节直接暴露给人工填写者，除非该信息是完成当前判断所必需的。

## 质量门规则

涉及工业级稳定性的步骤必须设置质量门：

1. 标注完成后，必须先生成 overlay 图并人工复查，再进入训练。
2. 模型训练前，必须确认数据集结构、标签格式、类别和坐标范围无错误。
3. 模型训练后，必须先检查预测可视化和错误样本，不只看单一指标。
4. OCR、VLM 或其他外部模型只作为证据分支时，必须保留原始输出和融合依据。
5. 若质量门不通过，必须回到上一步修正，不能用降低阈值或忽略异常来继续推进。

## 表格规则

需要人工填写的表格必须：

1. 只保留人工需要读、判断、填写的字段。
2. 把技术字段放到单独 JSON 或机器报告中。
3. 字段名使用任务语言，不使用内部变量名。
4. 默认按人工处理优先级排序。
5. 避免横向字段过多导致用户频繁左右滚动。
6. 对可选填写字段保持空白，不预填复杂提示文本。

## 图像排列规则

需要人工查看的图像页面必须：

1. 优先保证图纸主体足够大，方便看清标题栏。
2. 同一批任务按审核顺序稳定排列。
3. 每张图只显示必要的编号、样本名、候选位置、旋转角度等关键信息。
4. 图像网格应适合快速浏览和逐张放大，不应把过多调试信息挤在图像旁边。
5. 对比任务应把需要比较的图像靠近放置，减少来回跳转。
6. 大图优先使用 `object-fit: contain`，避免裁切标题栏。

## 标注任务规则

标注任务必须：

1. 明确告诉用户只需要标注什么。
2. 不要求用户理解模型训练细节。
3. 以最少类别开始，当前 YOLO/OBB 标题栏实验只使用 `title_block`。
4. 给出样本顺序和本地查看入口。
5. 标注后先生成 overlay 图供人工确认，再进入训练。

## 输出分层

人工输出和机器输出必须分层：

- 人工填写文件：少字段、低噪声、可直接操作。
- 机器报告文件：完整字段、调试信息、路径、置信度、错误码。
- 规则文件：长期约束，供后续生成工具遵守。

若人工界面和机器报告发生冲突，优先保证人工界面简洁，机器信息另存。
